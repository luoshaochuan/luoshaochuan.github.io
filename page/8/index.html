单调栈

单调栈具有如下性质：

  • 若是单调递增栈，则从栈顶到栈底的元素是严格递增的；若是单调递减栈，则从栈顶到栈底的元素是严格递减的。
  • 对于单调递增栈，若当前进栈元素比栈顶元素大(或相等)，则应将栈顶元素出栈，直至进栈元素比栈顶元素小；单调递减栈同理。
  • 单调递增栈栈顶元素左边保存的是数组左边第一个比栈顶元素大的元素。

1.右侧第1个比自己大的数

给一个数组，返回一个大小相同的数组。返回的数组的第i个位置的值应当是，对于原数组中的第i个元素，至少往右走多少步，才能遇到一个比自己大的元素（如果之后没有比自己大的元素，或者已经是最后一个元素，则在返回数组的对应位置放上-1）。
示例：
input:[5 3 1 2 4]
output:[-1 3 1 1 -1]

分析

维护一个单调递增栈，栈内保存数组元素下标，对于一个进栈元素，若比当前栈顶下标对应的元素大，说明该元素是比栈顶下标对应元素大的第一个数，则对应位置结果为二者下标差，将栈顶元素出栈，继续比较进栈元素和进栈元素大小，直至栈顶元素比进栈元素大，说明栈顶元素之前的元素都比进栈元素大。

排序算法小结

1.插入排序

(1)直接插入排序

基本思想：
对于未排序的数据，在已排序序列中从后向前扫描，找到相应位置插入。

算法描述：

  1. 从第一个元素开始，该元素可以被认为已排序
  2. 取出下一个元素，在已排序的序列中从后向前扫描
  3. 如果该元素(已排序)大于新元素，将该元素移到下一位置
  4. 重复步骤3，直到找到已排序的元素小于或者等于新元素的位置
  5. 将新元素插入到该位置后
  6. 重复步骤2~5

(2)希尔排序

基本思想：
希尔排序又称“缩小增量排序”，该方法的基本思想是：先将整个待排元素序列分割成若干个子序列（由相隔某个“增量”的元素组成的）分别进行直接插入排序，然后依次缩减增量再进行排序，待整个序列中的元素基本有序（增量足够小）时，再对全体元素进行一次直接插入排序。

算法描述：

先取一个正整数d1<n,把所有序号相隔d1的元素放入一组，组内进行直接插入排序，然后取d2<d1,重复上述分组和排序操作，直至di=1,即所有记录放进一个组中排序为止。

2.冒泡排序

基本思想：
不断比较再交换相邻元素

算法描述：

  1. 比较相邻的元素，如果前一个比后一个大，就把它们两个调换位置。
  2. 对每一对相邻元素作同样的工作，从开始第一对到结尾的最后一对。这步做完后，最后的元 素会是最大的数。
  3. 针对所有的元素重复以上的步骤，除了最后一个。
  4. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

冒泡排序改进：

对冒泡排序常见的改进方案是加一个标识变量flag，用于记录某趟扫描是否有交换操作，如果没有，说明数据已经排好序，算法可以终止了。

3.选择排序

基本思想：
每次找一个最小值。

算法描述：

  1. 找到最小元素，将该元素与第一个元素互换位置。
  2. 找到第二小元素，将该元素与第二个元素互换位置。
  3. 依次类推。

4.归并排序

基本思想：
归并（Merge）排序法是将两个（或两个以上）有序表合并成一个新的有序表，即把待排序序列分为若干个子序列，每个子序列是有序的。然后再把有序子序列合并为整体有序序列。

算法描述：
归并排序算法主要依赖归并(Merge)操作。归并操作指的是将两个已经排序的序列合并成一个序列的操作,归并操作步骤如下：

  1. 申请空间，使其大小为两个已经排序序列之和，该空间用来存放合并后的序列
  2. 设定两个指针，最初位置分别为两个已经排序序列的起始位置
  3. 比较两个指针所指向的元素，选择相对小的元素放入到合并空间，并移动指针到下一位置
  4. 重复步骤3直到某一指针到达序列尾
  5. 将另一序列剩下的所有元素直接复制到合并序列尾

5.堆排序

二叉堆的性质：

  1. 父节点的键值总是大于或等于(小于或等于)任何一个子节点的键值
  2. 每个节点的左子树和右子树都是一个二叉堆(都是最大堆或最小堆)

堆的存储：一般都用数组来表示堆，i节点的父节点为(i-1)/2,它的左右子节点下标分别为2i+1和2i+2.

基本思想：
利用完全二叉树中双亲节点和孩子节点之间的内在关系，在当前无序区中选择关键字最大(或者最小)的记录。也就是说，以最小堆为例，根节点为最小元素，较大的节点偏向于分布在堆底附近。

算法描述：

  1. 创建堆：从最后一个非叶节点num/2自底向下依次进行堆调整。
  2. 调整堆：对于每一节点，将其与左右子节点中最大的一个进行交换。
  3. 堆排序：从最后一个元素开始，依次与堆顶元素进行交换，每次交换后再进行堆调整。

6.快速排序

基本思想：
通过一趟排序将要排序的数据分割成两个独立的部分：分割点左边都是比它小的数，右边都是比它大的数。然后再按此方法对这两部分数据分别进行快速排序，整个排序过程可以递归进行，以此达到整个数据变成有序序列。

算法描述：

  1. 先从数组中选择一个数据作为基准数(通常是数组中的第一个元素)。
  2. 分别从数组前后进行扫描，设置两个变量i、j，排序开始的时候：i=0，j=N-1。
  3. 以第一个数组元素作为关键数据，赋值给key，即key=A[0]。
  4. 从j开始向前搜索，即由后开始向前搜索(j–)，找到第一个小于key的值A[j]，将A[j]和A[i]互换。
  5. 从i开始向后搜索，即由前开始向后搜索(i++)，找到第一个大于key的A[i]，将A[i]和A[j]互换。
  6. 重复第3、4步，直到i=j。

正则表达式基础

1.正则表达式概念

正则表达式描述了一种字符串匹配的模式，可以用来检查一个串是否含有某种子串、将匹配的子串做替换或者从某个串中取出符合某个条件的子串等。
正则表达式普通字符（例如字符 a 到 z）以及特殊字符（称为”元字符”）组成的文字模式。模式描述在搜索文本时要匹配的一个或多个字符串。正则表达式作为一个模板，将某个字符模式与所搜索的字符串进行匹配。

2.常用元字符
(1).字符处理
字符 描述
. 匹配换行符以外的任意字符
\w 匹配字母或数字或下划线或汉字
\s 匹配任意的空白字符，包括空格、制表符、换行符
\d 匹配数字
\b 匹配单词的开始或结尾
^ 匹配字符串的开头
$ 匹配字符串的结尾
[] 匹配中括号内任意一个字符

示例：
\bhello\b：匹配hello这个单词
^0\d\d-\d\d\d\d\d\d\d\d$：匹配一个例如025-87654321这样的字符串
[0-9]：匹配0-9中任意一个数字
[abcd]：匹配a,b,c,d中任意一个字符

(2).贪婪限定符

贪婪限定符用来指定正则表达式的一个给定组件必须要出现多少次才能满足匹配。有*或+或?或{n}或{n,}或{n,m}共6种。

字符 描述
* 重复前面的子表达式至少0次
+ 重复前面的子表达式至少1次
? 重复前面的子表达式0次或1次
{n} 重复前面的子表达式n次
{n,} 重复前面的子表达式至少n次
{n,m} 重复前面的子表达式n到m次

示例：
2\d+：2后面跟至少1个数字，例如20a、201a
0\d{2}-\d{8}：匹配025-87654321这样的一串数字

(3).非贪婪限定符

贪婪限定符会尽可能匹配更多的字符，非贪婪限定符会尽可能匹配更少的字符。

字符 描述
*? 匹配任意次，但尽可能少重复
+? 匹配至少1次，但尽可能少重复
?? 匹配0或1次，但尽可能少重复
{n,} 匹配至少n次，但尽可能少重复
{n.m} 匹配n到m次，但尽可能少重复
(4).反义符

有时候需要匹配不在某一范围的字符，这时候需要用到反义符。

字符 描述
\W 匹配任意不是字母、数字、下划线或汉字的字符
\S 匹配任意不是空白符的字符
\D 匹配任意非数字的字符
\B 匹配不是单词开头或结束的位置
[^x] 匹配除x外的任意字符
[^aoeiu] 匹配除aoeiu外的字符
(5).分支

可以用|来匹配它前面或后面任意一个正则表达式

(6).后向引用

后向引用用于重复搜索前面某个分组匹配的文本。使用小括号()括起来一个表达式后，将导致相关匹配存储到一个临时缓冲区中，后面可以用\n引用这个分组，n为组号，即每个分组从左到右出现的顺序。
你也可以用?<name>exp这种形式自己给分组命名，后面通过\k<name>后向引用即可。

Python正则表达式

Python提供一个re模块来处理正则表达式，常用的函数有re.match、re.search、re.sub等，下面分别来介绍这几个函数。

(1).re.match

re.match(pattern, string, flags=0)
参数:
pattern:正则表达式
string:要匹配的字符串
flags:标志位，用于控制正则表达式的匹配方式，比如是否区分大小写，多行匹配等。
注意：re.match匹配字符串的开始，如果字符串开始不符合正则表达式，则匹配失败。

(2).re.search

re.search(pattern, string, flags=0)
参数：
pattern:正则表达式
string:要匹配的字符串
flags:标志位，用于控制正则表达式的匹配方式，比如是否区分大小写，多行匹配等。
注意：re.search匹配整个字符串，直到找到一个匹配。
