正则表达式基础

1.正则表达式概念

正则表达式描述了一种字符串匹配的模式，可以用来检查一个串是否含有某种子串、将匹配的子串做替换或者从某个串中取出符合某个条件的子串等。
正则表达式普通字符（例如字符 a 到 z）以及特殊字符（称为”元字符”）组成的文字模式。模式描述在搜索文本时要匹配的一个或多个字符串。正则表达式作为一个模板，将某个字符模式与所搜索的字符串进行匹配。

2.常用元字符
(1).字符处理
字符 描述
. 匹配换行符以外的任意字符
\w 匹配字母或数字或下划线或汉字
\s 匹配任意的空白字符，包括空格、制表符、换行符
\d 匹配数字
\b 匹配单词的开始或结尾
^ 匹配字符串的开头
$ 匹配字符串的结尾
[] 匹配中括号内任意一个字符

示例：
\bhello\b：匹配hello这个单词
^0\d\d-\d\d\d\d\d\d\d\d$：匹配一个例如025-87654321这样的字符串
[0-9]：匹配0-9中任意一个数字
[abcd]：匹配a,b,c,d中任意一个字符

(2).贪婪限定符

贪婪限定符用来指定正则表达式的一个给定组件必须要出现多少次才能满足匹配。有*或+或?或{n}或{n,}或{n,m}共6种。

字符 描述
* 重复前面的子表达式至少0次
+ 重复前面的子表达式至少1次
? 重复前面的子表达式0次或1次
{n} 重复前面的子表达式n次
{n,} 重复前面的子表达式至少n次
{n,m} 重复前面的子表达式n到m次

示例：
2\d+：2后面跟至少1个数字，例如20a、201a
0\d{2}-\d{8}：匹配025-87654321这样的一串数字

(3).非贪婪限定符

贪婪限定符会尽可能匹配更多的字符，非贪婪限定符会尽可能匹配更少的字符。

字符 描述
*? 匹配任意次，但尽可能少重复
+? 匹配至少1次，但尽可能少重复
?? 匹配0或1次，但尽可能少重复
{n,} 匹配至少n次，但尽可能少重复
{n.m} 匹配n到m次，但尽可能少重复
(4).反义符

有时候需要匹配不在某一范围的字符，这时候需要用到反义符。

字符 描述
\W 匹配任意不是字母、数字、下划线或汉字的字符
\S 匹配任意不是空白符的字符
\D 匹配任意非数字的字符
\B 匹配不是单词开头或结束的位置
[^x] 匹配除x外的任意字符
[^aoeiu] 匹配除aoeiu外的字符
(5).分支

可以用|来匹配它前面或后面任意一个正则表达式

(6).后向引用

后向引用用于重复搜索前面某个分组匹配的文本。使用小括号()括起来一个表达式后，将导致相关匹配存储到一个临时缓冲区中，后面可以用\n引用这个分组，n为组号，即每个分组从左到右出现的顺序。
你也可以用?<name>exp这种形式自己给分组命名，后面通过\k<name>后向引用即可。

Python正则表达式

Python提供一个re模块来处理正则表达式，常用的函数有re.match、re.search、re.sub等，下面分别来介绍这几个函数。

(1).re.match

re.match(pattern, string, flags=0)
参数:
pattern:正则表达式
string:要匹配的字符串
flags:标志位，用于控制正则表达式的匹配方式，比如是否区分大小写，多行匹配等。
注意：re.match匹配字符串的开始，如果字符串开始不符合正则表达式，则匹配失败。

(2).re.search

re.search(pattern, string, flags=0)
参数：
pattern:正则表达式
string:要匹配的字符串
flags:标志位，用于控制正则表达式的匹配方式，比如是否区分大小写，多行匹配等。
注意：re.search匹配整个字符串，直到找到一个匹配。
